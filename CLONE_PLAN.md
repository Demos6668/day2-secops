## 1. Stack profile of `day2.osint`

| Layer | Technology | Notes |
|---|---|---|
| Frontend | **React 19 + Vite 7** (`@workspace/airowire-news` package) | `vite.config.ts` uses `@vitejs/plugin-react`, dev port `5173`, proxy `/api` → `http://localhost:8081`, `/ws` → ws. Manual chunks split `charts` (recharts) and `ui` (framer-motion + radix). |
| Language | **TypeScript 5.9** | `tsconfig.json` extends `../../tsconfig.base.json`, `moduleResolution: "bundler"`, `allowImportingTsExtensions`, `noEmit`. Path alias `@/* → src/*`, `@assets → ../../attached_assets`. |
| Styling | **Tailwind v4** via `@tailwindcss/vite` | CSS-first config: `@import "tailwindcss"` + `@theme inline { … }` block in `src/index.css`. CSS variables in `:root` (dark) and `.light`. Fonts: Outfit (sans), Fira Code (mono). |
| Components | **shadcn/ui "new-york"** (cssVariables: true, baseColor: neutral) on top of Radix primitives | `src/components/ui/*.tsx` — 55 primitives already present (button, card, dialog, drawer, sheet, tabs, popover, tooltip, command palette, sonner toaster, sidebar, table, skeleton, kbd, etc.). |
| Icons | **lucide-react** + `react-icons` | Lucide is the workspace standard. |
| Charts | **recharts** | Manual-chunked. Used in `Dashboard/SeverityTrendChart.tsx` etc. |
| State / data | **TanStack Query v5** + **next-themes** (dark default, key `airowire-theme`) | No Zustand here; lightweight `useState` + custom hooks. Generated React Query client from OpenAPI via Orval lives in `lib/api-client-react`. |
| Routing | **wouter** | Lazy-loaded routes, `<RouteErrorBoundary>` wrapper, `<Suspense>` fallback `Loader`. |
| Forms | **react-hook-form + zod + @hookform/resolvers** | Available, lightly used. |
| Motion | **framer-motion 12** | Used in Sidebar collapse, drawer transitions. |
| Toaster | **sonner 2** | Mounted in `App.tsx`. |
| Command palette | **cmdk 1.1** | Available (Phase-4 search uses this). |
| HTML sanitize | **dompurify 3** | For rendering remote markup safely. |
| Date | **date-fns 3** | `lib/utils.ts` exports `formatRelative`. |
| Monorepo | **pnpm workspaces** | `pnpm-workspace.yaml` defines `artifacts/*`, `lib/*`, `lib/integrations/*`, `scripts`. **`minimumReleaseAge: 1440`** (1-day supply-chain delay) — keep this. Pnpm catalog pins React 19.1.0, Tailwind 4.1.14, etc. |
| Testing | vitest 2 + @testing-library/react + jsdom | Configured at `vitest.config.ts`. |
| Auth / tenancy | None at the React layer. There is a `useWorkspaces()` hook + `WorkspaceSidebar` that switches "workspace" context client-side. No multi-tenant server logic. | Matches the brief: workspace is purely config-driven. |

**Tower/OSINT widgets that are reusable for a security dashboard:**

- Severity color system + chips (`SeverityBadge`, `getSeverityToken`, `CvssChip`) — already pairs color with text label, which satisfies our "RAG must not be color-only" rule once we add an icon.
- Status pills + freshness/connection badges (`StatusStrip`, `RefreshCountdown`, `FeedStatus`).
- Drill-down patterns (`ThreatModal`, `AdvisoryDetail`, `RecentlyViewedDrawer`, `WorkspaceSidebar` slide-over).
- Filter system (`FilterSection`, `ActiveFilterBar`, `useFilterParams`).
- Dashboard primitives (`StatsCard`, `ActivityStream`, `SeverityTrendChart`, `ThreatMeter`, `IndiaStatsPanel`).
- A real `AppLayout` (sidebar + header + footer + bottom nav + drawer) at `components/layout/`.

No world map, no graph view present in `day2.osint`. We will need both for the abcl-secviz "Feasibility / Linking" module (Phase 4, item 5).
